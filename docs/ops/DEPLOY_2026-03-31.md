## Vercel `multicel`

Problema visto despues del deploy backend:

- `https://multicel.vercel.app/api/precios/listas?inactivas=1` -> `404`
- luego `502`

Causa real confirmada:

- Vercel tenia una rewrite externa mal apuntada a `72.60.14.52:3000`
- ese puerto no esta expuesto al exterior
- la API publica correcta del server es `http://72.60.14.52/api`

Rewrite correcta:

```json
{
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "http://72.60.14.52/api/$1"
    }
  ]
}
```

Reglas practicas:

- no usar `:3000`
- no dejar espacios antes de `http`
- redeployar Vercel despues de cambiar routing rules

Validacion final esperada:

- `https://multicel.vercel.app/api/setup/status` devuelve JSON del backend
- `https://multicel.vercel.app/api/precios/listas?inactivas=1` devuelve `401`

- Hostinger actual es `pm2` multi-instancia con router publico
